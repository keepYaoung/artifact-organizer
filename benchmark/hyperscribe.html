Artifact Organizer Codebase Map

How a semantic envelope becomes one self-contained HTML page

Overview

The model writes structure. The renderer turns that structure into a browser-ready artifact.

Artifact Organizer is a catalog-driven renderer. The important split is simple:

  • plugins/hyperscribe/spec/catalog.json defines the contract.
  • plugins/hyperscribe/scripts/render.mjs validates and renders.
  • plugins/hyperscribe/scripts/components/*.mjs turns semantic nodes into HTML.
  • plugins/hyperscribe/assets/* and plugins/hyperscribe/themes/* supply the shared look and behavior.
Key idea

The envelope carries meaning, not presentation. That keeps the output predictable and makes the HTML easy to regenerate from the same source JSON.

Render Pipeline

A single pass moves from prompt to validated envelope to static HTML.

flowchart TD
A["User prompt"] --> B["Command or skill prompt"] --> C["Semantic JSON envelope"] --> D["Schema validation<br/>spec/catalog.json"] --> E["Tree walk<br/>scripts/lib/tree.mjs"] --> F["Component SSR<br/>scripts/components/*.mjs"] --> G["Theme + asset bundle<br/>base.css + themes/*.css"] --> H["Self-contained HTML<br/>+ sibling JSON"]

Repository Shape

The product source lives under plugins/hyperscribe. The skill and docs folders are mirrors and helpers around it.

plugins/hyperscribe
Canonical runtime
  • Renderer, schema, tree walker, themes, assets, and commands live here.
  • This is the source of truth for the envelope contract and HTML output.
  • Most component behavior is in `scripts/components/*.mjs`.
source of truth
skills / docs / tests
Surrounding mirrors and checks
  • Skill bundles expose the same renderer to agents and command entry points.
  • Tests pin the contract with fixtures, validation, and golden output.
  • Docs explain the protocol; they do not define it.
